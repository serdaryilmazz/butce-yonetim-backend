Angular 19 (standalone) + Node.js (Express) + MongoDB (Mongoose) kullan.
Kod clean architecture, modüler ve scalable olsun.
TypeScript strict mode kullan.
Backend için controller-service pattern uygula.
🏗️ 1. BACKEND SETUP
Node.js + Express backend projesi oluştur.

Özellikler:
- Express server
- MongoDB bağlantısı (Mongoose)
- dotenv kullan
- CORS aktif
- JSON middleware

Klasör yapısı:

/src
  /config
  /models
  /routes
  /controllers
  /services
  /middlewares
  /utils

Server 3000 portunda çalışsın.
MongoDB bağlantısı .env'den alınsın.
🔌 2. DATABASE CONNECTION
Mongoose ile MongoDB bağlantısı kur.

- connectDB fonksiyonu yaz
- bağlantı başarılı / hatalı log bas
- app start olmadan önce DB bağlansın
🧱 3. MODELLER (DÜZELTİLMİŞ HAL)
Mongoose modellerini oluştur:

User:
- email (unique)
- password
- timestamps

Category:
- name
- type: 'income' | 'expense'
- icon
- isDefault
- userId (optional)
- timestamps

Transaction:
- userId (ZORUNLU)
- type
- amount
- categoryId (ref Category)
- date
- note
- timestamps

RecurringTransaction:
- userId
- amount
- categoryId
- interval: 'monthly'
- lastExecutedDate
- timestamps

Budget:
- userId
- categoryId
- limit
- timestamps
🔐 4. AUTH SYSTEM
JWT tabanlı auth sistemi kur:

- Register endpoint
- Login endpoint
- bcrypt ile password hash
- JWT token üret

Middleware:
- authMiddleware yaz
- token doğrula
- req.user içine userId ekle
🧾 5. TRANSACTION API (USER SCOPED)
Transaction API yaz:

- GET /transactions (sadece user’ın verisi)
- POST /transactions
- PUT /transactions/:id
- DELETE /transactions/:id

Kurallar:
- Her işlem req.user.id ile ilişkilendirilsin
- Başkasının datasına erişim olmasın
📁 6. CATEGORY API + DEFAULT SEED
Category API yaz:

- GET /categories
- POST /categories
- DELETE /categories/:id

Ekstra:
- Uygulama ilk başladığında default kategoriler oluştur:
  (Maaş, Burs, Kira, Yemek vb.)
- Default kategoriler global olsun
🔄 7. RECURRING SYSTEM (DÜZELTİLMİŞ)
Recurring transaction sistemi yaz:

- Uygulama her requestte kontrol etsin ama:
  - lastExecutedDate kontrolü yap
  - aynı ay tekrar eklenmesin

Gerekli transactionları otomatik oluştur.
📊 8. ANALYTICS API (GENİŞ)
Analytics endpointleri oluştur:

- toplam gider
- bakiye

GET /analytics/top-category

GET /analytics/monthly-comparison

GET /analytics/category-distribution
(pie chart için)

GET /analytics/daily-expenses
(line chart için)
🤖 9. AI ANALİZ SERVİSİ
Transaction verilerini analiz eden servis yaz:

Kurallar:
- Harcama artışı varsa tespit et
- En çok harcanan kategoriyi bul
- Gereksiz harcama patternlerini yakala

Endpoint:
GET /analytics/ai-insights

Response:
string[] öneriler
💰 10. BUDGET API
Budget sistemi yaz:

- POST /budgets
- GET /budgets

Ekstra:
- Harcama limitini aşınca flag üret
